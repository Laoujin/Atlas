Static site generators in 2026 for a personal notes site

TL;DR. Three strong options for a notes site in 2026: Astro, Hugo, 11ty. Default to Astro unless build-speed under a second matters [1] — then Hugo [2]. Pick 11ty only if you want zero JS on the page and full template control [3].

Comparison

Criterion Astro Hugo 11ty
Cold build, ~500 pages ~4 s [1] ~0.6 s [2] ~3 s [4]
Islands / partial hydration First-class [5] None Opt-in via `@11ty/is-land` [6]
Markdown → HTML remark + MDX built-in [7] Goldmark, very fast [8] markdown-it, swappable [9]
Learning curve Low if you know React/Vue [10] Medium — Go templates [11] Low-medium — Nunjucks/Liquid [12]
Plugin ecosystem Large, growing fast [13] Modest; often "done" with core Curated, small

Who says what

r/webdev consensus (2026 Q1): Astro for new projects; don't migrate a working Hugo site just to chase DX [10].
GitHub stars, Apr 2026: Hugo 78k, Astro 52k, 11ty 18k [14]. Growth rate favours Astro (≈ 15 k/yr vs Hugo ≈ 6 k/yr).

Recommendation for a personal notes site

Astro. Notes sites are read-heavy, lightly interactive (search, ToC). Astro's islands let you drop in a small search widget without shipping a whole framework, and Markdown/MDX support is batteries-included [7]. Choose Hugo if you have >5 000 notes and build speed drives authoring flow [2].
